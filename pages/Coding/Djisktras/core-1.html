Section 01

📌 Problem Statement

Given an m × n grid with:

H= Hospital
X= House
.= Road (passable)
#= Wall (impassable)

Each passable cell has a travel cost. For every house, find the shortest distance to the nearest hospital.

Section 03

📌 Core Intuition

❌ Naive Thinking

Run Dijkstra from each hospital separately, then for each house pick the minimum.

If there are K hospitals → K separate Dijkstra runs
→ K × O((V+E) log V)

✅ Multi-Source Thinking

Push ALL hospitals into the heap at distance 0 simultaneously. Run Dijkstra ONCE. Water pours from all hospitals at the same time — whichever reaches a house first wins.

Just ONE Dijkstra run → O((V+E) log V)

The Mental Model

Imagine a FAKE super-node connected to all hospitals with cost 0:

DIAGRAMSuper Source Concept
                 SUPER SOURCE
                /   |   \
          cost=0  cost=0  cost=0
              /     |       \
        Hosp A   Hosp B   Hosp C
           \       |        /
            \      |       /
             actual grid roads
✅
You don't build this in code. Just initialize all hospitals at distance 0 in the heap. Same effect!
Section 04

📌 Algorithm Steps

  1. 1Parse grid → find all hospitals and houses
  2. 2Initialize dist matrix with ∞ everywhere
  3. 3Set dist = 0 for ALL hospitals, push them into min-heap
  4. 4Run standard Dijkstra
  5. 5Read off dist values for each house
Section 05

📌 Core Solution Code

Python 
import heapq

def nearest_hospital(grid, cost):
    m, n = len(grid), len(grid[0])
    INF = float('inf')
    dist = [[INF] * n for _ in range(m)]
    heap = []

    # ---- Multi-Source: Push ALL hospitals at distance 0 ----
    for r in range(m):
        for c in range(n):
            if grid[r][c] == 'H':
                dist[r][c] = 0
                heapq.heappush(heap, (0, r, c))

    # ---- Standard Dijkstra ----
    dirs = [(0, 1), (0, -1), (1, 0), (-1, 0)]

    while heap:
        d, r, c = heapq.heappop(heap)

        # Skip stale entries
        if d > dist[r][c]:
            continue

        for dr, dc in dirs:
            nr, nc = r + dr, c + dc
            if 0 <= nr < m and 0 <= nc < n and grid[nr][nc] != '#':
                new_dist = d + cost[nr][nc]
                if new_dist < dist[nr][nc]:
                    dist[nr][nc] = new_dist
                    heapq.heappush(heap, (new_dist, nr, nc))

    # ---- Collect answers for houses ----
    result = {}
    for r in range(m):
        for c in range(n):
            if grid[r][c] == 'X':
                result[(r, c)] = dist[r][c] if dist[r][c] != INF else -1

    return result

Complexity

Time
O(mn·log(mn))
Space
O(m × n)
Section 06

📌 How to Read the Distance Matrix

After running the algorithm, dist looks like:

col→ 0 1 2 3 4 Row 0: [ 0, 2, 5, #, 6 ] Row 1: [ 4, #, 4, 6, 3 ] Row 2: [ 6, 5, 4, 2, 0 ] Row 3: [ 9, #, 8, 3, 2 ] Row 4: [10, 14, 10, #, 5 ]

How to read: dist[r][c] = cheapest cost to travel from cell (r,c) to the nearest hospital.

🏠 House at (2,2): dist[2][2] = 4 → nearest hospital is 4 cost away

🏠 House at (4,0): dist[4][0] = 10 → nearest hospital is 10 cost away

Section 07

📌 Follow-Up 1: Track Which Hospital is Nearest

🔍
Problem Extension: Not just the distance, but also which hospital is closest to each house.

Approach

Maintain a nearest matrix alongside dist. When you update a cell's distance, also copy over which hospital it was reached from.

Python 
import heapq

def nearest_hospital_with_tracking(grid, cost):
    m, n = len(grid), len(grid[0])
    INF = float('inf')
    dist    = [[INF]  * n for _ in range(m)]
    nearest = [[None] * n for _ in range(m)]
    heap = []

    # ---- Multi-Source: Each hospital knows itself ----
    for r in range(m):
        for c in range(n):
            if grid[r][c] == 'H':
                dist[r][c] = 0
                nearest[r][c] = (r, c)  # I am my own nearest hospital
                heapq.heappush(heap, (0, r, c))

    # ---- Dijkstra ----
    dirs = [(0, 1), (0, -1), (1, 0), (-1, 0)]

    while heap:
        d, r, c = heapq.heappop(heap)

        if d > dist[r][c]:
            continue

        for dr, dc in dirs:
            nr, nc = r + dr, c + dc
            if 0 <= nr < m and 0 <= nc < n and grid[nr][nc] != '#':
                new_dist = d + cost[nr][nc]
                if new_dist < dist[nr][nc]:
                    dist[nr][nc]    = new_dist
                    nearest[nr][nc] = nearest[r][c]  # Inherit parent's hospital
                    heapq.heappush(heap, (new_dist, nr, nc))

    # ---- Results ----
    result = {}
    for r in range(m):
        for c in range(n):
            if grid[r][c] == 'X':
                result[(r, c)] = {
                    'distance': dist[r][c] if dist[r][c] != INF else -1,
                    'nearest_hospital': nearest[r][c]
                }

    return result

Section 08

📌 Follow-Up 2: Trace the Actual Path

🔍
Problem Extension: Not just distance and which hospital — show the exact path from the house to its nearest hospital.

Approach

Maintain a parent matrix. When cell (nr, nc) is updated from cell (r, c), record parent[nr][nc] = (r, c). Then backtrack from house to hospital.

Part A — Modified Dijkstra with Parent Tracking

Python 
import heapq

def dijkstra_with_path(grid, cost):
    m, n = len(grid), len(grid[0])
    INF = float('inf')
    dist   = [[INF]  * n for _ in range(m)]
    parent = [[None] * n for _ in range(m)]  # NEW: track parent cell
    heap = []

    # ---- Multi-Source ----
    for r in range(m):
        for c in range(n):
            if grid[r][c] == 'H':
                dist[r][c] = 0
                parent[r][c] = None  # Hospital has no parent (it's the root)
                heapq.heappush(heap, (0, r, c))

    # ---- Dijkstra ----
    dirs = [(0, 1), (0, -1), (1, 0), (-1, 0)]

    while heap:
        d, r, c = heapq.heappop(heap)

        if d > dist[r][c]:
            continue

        for dr, dc in dirs:
            nr, nc = r + dr, c + dc
            if 0 <= nr < m and 0 <= nc < n and grid[nr][nc] != '#':
                new_dist = d + cost[nr][nc]
                if new_dist < dist[nr][nc]:
                    dist[nr][nc]   = new_dist
                    parent[nr][nc] = (r, c)  # NEW: remember who updated me
                    heapq.heappush(heap, (new_dist, nr, nc))

    return dist, parent

Part B — Path Reconstruction Function

Python 
def reconstruct_path(parent, house_r, house_c):
    """
    Backtrack from house to hospital using parent pointers.
    Returns path from HOSPITAL → HOUSE (forward direction).
    """
    path = []
    r, c = house_r, house_c

    while (r, c) is not None:
        path.append((r, c))
        if parent[r][c] is None:
            break  # Reached the hospital (root)
        r, c = parent[r][c]

    path.reverse()  # Reverse: hospital → house
    return path

Part C — Putting It All Together

Python 
# Run Dijkstra
dist, parent = dijkstra_with_path(grid, cost)

# For each house, get path
for r in range(len(grid)):
    for c in range(len(grid[0])):
        if grid[r][c] == 'X':
            path = reconstruct_path(parent, r, c)
            print(f"House ({r},{c}):")
            print(f"  Distance = {dist[r][c]}")
            print(f"  Path: {' → '.join(str(p) for p in path)}")

Section 09

📌 Follow-Up 3: Unweighted Grid (All Costs = 1)

💬
Interviewer Says: "What if all road costs are equal (say 1)?"
Answer: Use plain BFS instead of Dijkstra. No heap needed. Faster.
Python 
from collections import deque

def nearest_hospital_bfs(grid):
    m, n = len(grid), len(grid[0])
    INF = float('inf')
    dist  = [[INF] * n for _ in range(m)]
    queue = deque()

    # ---- Multi-Source BFS ----
    for r in range(m):
        for c in range(n):
            if grid[r][c] == 'H':
                dist[r][c] = 0
                queue.append((r, c))

    dirs = [(0, 1), (0, -1), (1, 0), (-1, 0)]

    while queue:
        r, c = queue.popleft()
        for dr, dc in dirs:
            nr, nc = r + dr, c + dc
            if (0 <= nr < m and 0 <= nc < n
                    and grid[nr][nc] != '#'
                    and dist[nr][nc] == INF):
                dist[nr][nc] = dist[r][c] + 1
                queue.append((nr, nc))

    return dist

Complexity

Time
O(m × n)
Space
O(m × n)
Advantage
No log factor!
Section 10

📌 Follow-Up 4: Costs are Only 0 or 1

💬
Interviewer Says: "What if travel costs are either 0 or 1?"
Answer: Use 0-1 BFS with deque. Push cost-0 edges to front, cost-1 edges to back.
Python 
from collections import deque

def nearest_hospital_01bfs(grid, cost):
    m, n = len(grid), len(grid[0])
    INF = float('inf')
    dist = [[INF] * n for _ in range(m)]
    dq   = deque()

    # ---- Multi-Source ----
    for r in range(m):
        for c in range(n):
            if grid[r][c] == 'H':
                dist[r][c] = 0
                dq.appendleft((r, c))

    dirs = [(0, 1), (0, -1), (1, 0), (-1, 0)]

    while dq:
        r, c = dq.popleft()

        for dr, dc in dirs:
            nr, nc = r + dr, c + dc
            if 0 <= nr < m and 0 <= nc < n and grid[nr][nc] != '#':
                w = cost[nr][nc]  # 0 or 1
                new_dist = dist[r][c] + w
                if new_dist < dist[nr][nc]:
                    dist[nr][nc] = new_dist
                    if w == 0:
                        dq.appendleft((nr, nc))  # Free → front
                    else:
                        dq.append((nr, nc))       # Cost 1 → back

    return dist
Time
O(m × n)
Space
O(m × n)
Advantage
No log factor!

📌 Quick Decision Table

Which algorithm to use based on edge weight characteristics:

Edge Weights Algorithm Data Structure Time
All equal (unweighted) BFS Queue (deque) O(V + E)
Only 0 and 1 0-1 BFS Deque O(V + E)
Variable non-negative Dijkstra Min-Heap O((V+E) log V)
Negative weights exist Bellman-Ford Array O(V × E)
Section 12

📌 Edge Cases to Remember

🧱 House surrounded by walls
dist stays ∞ → return -1
🏥 No hospitals in grid
All distances stay ∞ → return -1 for all
📍 Hospital right next to house
Distance = cost of house cell (1 step away)
🔄 House IS on a hospital cell
Can't happen — different cell types by definition
📦 Grid is 1×1 with only a house
No hospital exists → return -1
⚖️ Multiple paths with same cost
Dijkstra picks any one — all are equally optimal
