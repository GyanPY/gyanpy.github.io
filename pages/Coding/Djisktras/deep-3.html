§10Algorithm Selection

The Decision Tree

Every shortest-path problem walks down the same three questions: are weights given, what kind, and what's the cost structure? Run that tree in your head on the first read of the problem — by the time you're done parsing the input, you should already know the algorithm.

§11Common Patterns & Tricks

The Five Tricks That Show Up Everywhere

These show up in nearly every Dijkstra problem regardless of variation. Drilling them into muscle memory is what turns a 30-minute coding session into a 10-minute one.

1 Early Termination

When you only need the distance to one target, return the moment it's popped from the heap. Greedy guarantee says: that's the final shortest distance.

Python · Early Exit 
# When you only need distance to ONE target:
if u == target:
    return d  # STOP HERE — guaranteed shortest!
2 Stale Entry Skip

Always include this check immediately after popping. The heap can hold outdated entries (a better path was found before this one came up); skipping them is essentially free and prevents wasted relaxations.

Python · The Critical Skip 
# ALWAYS include this in Dijkstra:
if d > dist[u]:
    continue  # This entry is outdated — a better path was found
3 Grid as Graph

Cells are nodes, adjacent cells are edges. Memorise dirs = [(0,1),(0,-1),(1,0),(-1,0)] and the bounds-check pattern — they appear in roughly half of L5 graph problems.

Python · Grid Neighbour Iteration 
# Grid problems: cells are nodes, adjacent cells are edges
dirs = [(0,1), (0,-1), (1,0), (-1,0)]
for dr, dc in dirs:
    nr, nc = r+dr, c+dc
    if 0 <= nr < m and 0 <= nc < n:
        # Process neighbor (nr, nc)
4 Dictionary vs Array for dist

Pick the container that fits the state space. Arrays are faster when you have dense integer-indexed nodes; dictionaries are necessary when state keys are tuples, strings, or sparse.

Python · Choosing the Container 
# ARRAY: when nodes are 0 to N-1 (dense)
dist = [float('inf')] * n

# 2D ARRAY: when grid
dist = [[float('inf')]*n for _ in range(m)]

# DICTIONARY: when states are sparse/complex (implicit graphs)
dist = {}  # dist[state] = cost
5 Multiple Dimensions in dist

State Expansion (Variation 2) lives or dies on this one. The number of dimensions follows the constraints — one extra constraint = one extra dimension on dist.

Python · Dimensional Templates 
# 1D: Standard
dist[node]

# 2D: With one constraint
dist[node][stops]
dist[node][fuel]

# 3D: Grid with constraint
dist[row][col][removals]

# Bitmask: Keys/items collected
dist[row][col][keys_bitmask]
§12Interview Execution Framework

The Six Steps You Walk Out Loud

Run these in order. The first three are pre-coding (mostly verbal — communicating your thinking is half the score), the next three are coding and verification. Don't skip steps even if you "see the answer" — the interviewer is grading the walk, not just the destination.

📌Step-By-Step When You See a Shortest-Path Problem

1
Step 1
IDENTIFY
  • Is this a shortest-path problem?
  • What are nodes? What are edges? What are weights?
2
Step 2
CLASSIFY
  • Which variation? (Standard / State Expansion / Multi-Source / etc.)
  • What are edge weights? (Equal / 0-1 / Variable)
3
Step 3
SELECT ALGORITHM

Use the decision tree from §10 above. Justify your pick out loud — "weights vary so heap-based Dijkstra; if they were 0/1 I'd switch to a deque for an O(V+E) win."

4
Step 4
CODE
  • Use the appropriate template (don't reinvent — recall)
  • Handle edge cases as you go
5
Step 5
ANALYZE
  • Time complexity
  • Space complexity
  • Can it be optimized? (e.g., 0-1 BFS instead of Dijkstra?)
6
Step 6
TEST
  • Small example dry run
  • Edge cases: empty graph, single node, unreachable target

📌What Google L5 Interviewers Look For

The skill column is what they're scoring. The "How They Test" column is what they say or do to probe it. Knowing both lets you actively serve up the signal instead of waiting to be tested.
