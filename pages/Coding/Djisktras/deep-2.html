Part 1 covered the algorithm and its first four variations. Part 2 takes on the three that interviewers love because they look like new problems but aren't: Minimax changes the relaxation operator from + to max and unlocks bottleneck problems; K-th Shortest changes the visited-once invariant to visited-K-times; and Implicit Graph changes nothing about the algorithm at all — it just changes where the graph comes from. The trick at every L5 graph problem is recognising which of these you're staring at.

§7Variation 5

Minimax Dijkstra

05
Variation 5 · Change the operator, change the meaning
Minimax Dijkstra
new = max(dist, weight) · only ONE line changes

📌Problem Signal

Recognize this when you see…

"Minimize the maximum edge", "steepest climb", "bottleneck", "what time can you first reach".

📌Core Idea

The whole change · one line
Normal:  new_dist = dist + weight
Minimax:  new_dist = max(dist, weight)
From "sum of edges" to "worst single edge". Same algorithm, different optimization target.

📌Template

Python · Minimax Dijkstra 
import heapq

def minimax_dijkstra(graph, source, target):
    dist = {node: float('inf') for node in graph}
    dist[source] = 0
    heap = [(0, source)]

    while heap:
        max_so_far, u = heapq.heappop(heap)
        if u == target:
            return max_so_far
        if max_so_far > dist[u]:
            continue
        for v, weight in graph[u]:
            new_max = max(max_so_far, weight)  # THE CHANGE
            if new_max < dist[v]:
                dist[v] = new_max
                heapq.heappush(heap, (new_max, v))
    return -1

📌Grid Template — Minimum Effort Path

Python · Path of Minimum Effort 
import heapq

def minimum_effort_path(heights):
    m, n = len(heights), len(heights[0])
    INF = float('inf')
    dist = [[INF]*n for _ in range(m)]
    dist[0][0] = 0
    heap = [(0, 0, 0)]
    dirs = [(0,1),(0,-1),(1,0),(-1,0)]

    while heap:
        effort, r, c = heapq.heappop(heap)
        if r == m-1 and c == n-1:
            return effort
        if effort > dist[r][c]:
            continue
        for dr, dc in dirs:
            nr, nc = r+dr, c+dc
            if 0 <= nr < m and 0 <= nc < n:
                edge = abs(heights[nr][nc] - heights[r][c])
                new_effort = max(effort, edge)
                if new_effort < dist[nr][nc]:
                    dist[nr][nc] = new_effort
                    heapq.heappush(heap, (new_effort, nr, nc))

    return dist[m-1][n-1]

📌Maximin — Reverse · Maximize the Minimum

The mirror problem

If minimax = "minimise the worst edge", maximin = "maximise the worst edge". Use a max-heap (negate values) and flip the relaxation comparison.

Python · Maximin Path 
import heapq

def maximin_path(graph, source, target):
    dist = {node: -1 for node in graph}
    dist[source] = float('inf')
    heap = [(-float('inf'), source)]  # MAX-heap via negation

    while heap:
        neg_min, u = heapq.heappop(heap)
        min_edge = -neg_min
        if u == target:
            return min_edge
        if min_edge < dist[u]:
            continue
        for v, weight in graph[u]:
            new_min = min(min_edge, weight)
            if new_min > dist[v]:  # LARGER is better
                dist[v] = new_min
                heapq.heappush(heap, (-new_min, v))
    return -1

📌Alternative — Binary Search + BFS

For minimax problems with a known weight set, you can binary-search the answer instead. Same answer, different mechanic — useful when interviewer asks "any other way?".

Python · Binary Search Approach 
def minimax_binary_search(graph, source, target, all_weights):
    weights = sorted(set(all_weights))
    left, right = 0, len(weights) - 1
    answer = weights[-1]

    while left <= right:
        mid = (left + right) // 2
        if can_reach_with_threshold(graph, source, target, weights[mid]):
            answer = weights[mid]
            right = mid - 1
        else:
            left = mid + 1
    return answer

§8Variation 6

K-th Shortest Path

06
Variation 6 · Break the visited-once invariant
K-th Shortest Path
count[node] += 1 · pop each node up to K times

📌Problem Signal

Recognize this when you see…

"2nd shortest", "K-th cheapest", "alternative route", "backup path".

📌Core Idea

From "once" to "up to K times"
Normal Dijkstra:  Pop each node ONCE  →  1st shortest
K-th Shortest:  Pop each node up to K times  →  K-th shortest
CRITICAL guard: if count[node] > K: continue — prevents infinite loops from cycles!

📌K-th Shortest Template

Python · K-th Shortest Path 
import heapq
from collections import defaultdict

def kth_shortest(graph, source, target, k):
    count = defaultdict(int)
    heap = [(0, source)]

    while heap:
        dist, node = heapq.heappop(heap)
        count[node] += 1

        if node == target and count[node] == k:
            return dist

        if count[node] > k:
            continue

        for neighbor, weight in graph[node]:
            heapq.heappush(heap, (dist + weight, neighbor))

    return -1

📌Second Shortest (Strictly Greater) — Two Distances Per Node

For the special case K = 2 where the second-shortest must be strictly greater than the first, track two distances per node. Faster than the generic K-th template, and a common interview ask.

Python · Second Shortest Path 
import heapq
from collections import defaultdict

def second_shortest(n, edges, source, target):
    graph = defaultdict(list)
    for u, v, w in edges:
        graph[u].append((v, w))
        graph[v].append((u, w))

    INF = float('inf')
    dist1 = [INF] * (n + 1)
    dist2 = [INF] * (n + 1)
    dist1[source] = 0
    heap = [(0, source)]

    while heap:
        d, u = heapq.heappop(heap)
        if d > dist2[u]:
            continue
        for v, w in graph[u]:
            nd = d + w
            if nd < dist1[v]:
                dist2[v] = dist1[v]
                dist1[v] = nd
                heapq.heappush(heap, (nd, v))
            elif dist1[v] < nd < dist2[v]:
                dist2[v] = nd
                heapq.heappush(heap, (nd, v))

    return dist2[target]

📌Key Distinctions

Three different "K-th" interpretations — interviewers will ask follow-ups to test if you noticed which one they meant. Pin down the definition before coding.

K-th shortest PATH
K-th path overall — same distance values are allowed (multiple paths can tie).
K-th DISTINCT distance
Strictly increasing distances — only count unique distance values.
K-th SIMPLE path
No node revisits allowed → use Yen's Algorithm, not standard Dijkstra.
§9Variation 7

Implicit Graph

07
Variation 7 · The graph isn't given — you generate it
Implicit Graph
get_neighbors(state) · transitions defined on-the-fly

📌Problem Signal

Recognize this when you see…

"Minimum operations to transform", "puzzle / game", "no explicit graph", "state space".

📌Core Idea — Six Steps

State
Define what counts as a node — a position, a board, a tuple.
Transitions
Define edges — which states reachable from the current one?
Costs
Define edge weights — usually 1 (BFS) or variable (Dijkstra).
Target
Define the goal state — when do we stop?
Prune
Cut search space aggressively — avoid infinite exploration.
Choose
BFS / 0-1 BFS / Dijkstra / A* — based on cost structure.

📌Generic Template

Python · Implicit Dijkstra 
import heapq

def implicit_dijkstra(start, is_target, get_neighbors):
    dist = {start: 0}
    heap = [(0, start)]

    while heap:
        cost, state = heapq.heappop(heap)
        if is_target(state):
            return cost
        if cost > dist.get(state, float('inf')):
            continue
        for next_state, edge_cost in get_neighbors(state):
            new_cost = cost + edge_cost
            if new_cost < dist.get(next_state, float('inf')):
                dist[next_state] = new_cost
                heapq.heappush(heap, (new_cost, next_state))
    return -1

📌BFS Version — All Costs Equal

When all edge costs are equal
BFS beats Dijkstra · O(V + E) instead of O((V+E) log V)
Drop the heap, use a deque. The lower complexity is "free" once you spot equal costs.
Python · Implicit BFS 
from collections import deque

def implicit_bfs(start, is_target, get_neighbors):
    visited = {start}
    queue = deque([(start, 0)])

    while queue:
        state, steps = queue.popleft()
        if is_target(state):
            return steps
        for next_state, _ in get_neighbors(state):
            if next_state not in visited:
                visited.add(next_state)
                queue.append((next_state, steps + 1))
    return -1

📌Pruning Strategies

Implicit graphs can be infinite or astronomically large. Pruning isn't optimization — it's the difference between "answer in 50ms" and "memory error".

Strategy When Example
Upper / lower boundNumeric states0 ≤ num ≤ target × 2
Bounding boxCoordinate statesabs(x) ≤ limit
State space sizePuzzles2×3 puzzle has only 720 states
Known upper boundAnyIf BFS found answer = 20, prune cost > 20

📌State Must Be Hashable!

Python · The Trap vs The Fix 
# ❌ WRONG
state = [1, 2, 3, 0]      # Lists CANNOT be dict keys
visited.add(state)         # TypeError!

# ✅ CORRECT
state = tuple([1, 2, 3, 0])  # Tuples CAN be dict keys
visited.add(state)             # Works! ✅
Why this trips senior candidates

Lists are mutable, so Python refuses to hash them. The moment you try visited.add(some_list) or dist[some_list], you get a TypeError: unhashable type. Always convert states to tuples (or frozen sets) before they enter your visited set or distance dictionary.
