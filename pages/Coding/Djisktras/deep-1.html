§1Core Algorithm

The Master Algorithm

📌What is Dijkstra?

A greedy algorithm that finds the shortest path from a source to all other nodes in a weighted graph with non-negative edge weights.

The One Analogy

The water analogy
Water poured from source. Flows to the nearest point first, then gradually spreads. The first time water reaches any node = the shortest path.

📌Core Template — Memorize This

Python · Master Template 
import heapq

def dijkstra(graph, source):
    dist = {node: float('inf') for node in graph}
    dist[source] = 0
    heap = [(0, source)]

    while heap:
        d, u = heapq.heappop(heap)

        if d > dist[u]:  # Skip stale entries
            continue

        for v, weight in graph[u]:
            new_dist = d + weight
            if new_dist < dist[v]:
                dist[v] = new_dist
                heapq.heappush(heap, (new_dist, v))

    return dist

Complexity

Time
O((V + E) log V)
Space
O(V + E)

📌Core Properties

Greedy guarantee
Once a node is popped from the heap, its distance is FINALIZED.
Non-negative only
Negative edges break Dijkstra.
Early termination
If you only need dist to ONE target, return when target is popped.
Stale entry skip
if d > dist[u]: continue — critical optimization.

§3Variation 1

Standard Dijkstra

01
Variation 1 · The baseline
Standard Dijkstra
new = dist + weight · the change-line is just addition

📌Problem Signal

Recognize this when you see…

"Find shortest / cheapest path", weighted graph, non-negative weights.

📌Grid Version

Python · Dijkstra on Grid 
import heapq

def dijkstra_grid(grid):
    m, n = len(grid), len(grid[0])
    dist = [[float('inf')] * n for _ in range(m)]
    dist[0][0] = grid[0][0]
    heap = [(grid[0][0], 0, 0)]
    dirs = [(0,1), (0,-1), (1,0), (-1,0)]

    while heap:
        d, r, c = heapq.heappop(heap)
        if r == m-1 and c == n-1:
            return d
        if d > dist[r][c]:
            continue
        for dr, dc in dirs:
            nr, nc = r+dr, c+dc
            if 0 <= nr < m and 0 <= nc < n:
                nd = d + grid[nr][nc]
                if nd < dist[nr][nc]:
                    dist[nr][nc] = nd
                    heapq.heappush(heap, (nd, nr, nc))

    return dist[m-1][n-1]

📌With Path Reconstruction

Python · Dijkstra + Parent Tracking 
import heapq

def dijkstra_with_path(graph, source, target):
    dist = {node: float('inf') for node in graph}
    parent = {node: None for node in graph}
    dist[source] = 0
    heap = [(0, source)]

    while heap:
        d, u = heapq.heappop(heap)
        if u == target:
            break
        if d > dist[u]:
            continue
        for v, w in graph[u]:
            if d + w < dist[v]:
                dist[v] = d + w
                parent[v] = u
                heapq.heappush(heap, (d + w, v))

    # Reconstruct path
    path = []
    node = target
    while node is not None:
        path.append(node)
        node = parent[node]
    path.reverse()

    return dist[target], path
§4Variation 2

State Expansion

02
Variation 2 · The L5 differentiator
State Expansion
dist[node][state] · add a dimension to the state

📌Problem Signal

Recognize this when you see…

Extra constraint: "at most K stops", "budget limit", "collect keys", "K obstacles removed".

📌Core Idea

Normal:   dist[node]            = shortest distance
Expanded: dist[node][constraint_value] = shortest distance with THIS constraint

State becomes: (node, constraint_remaining)

📌When To Use

Problem Clue Extra State Dimension
"At most K obstacles removed"remaining_removals
"At most K stops"stops_used
"Budget / fuel constraint"budget_remaining
"Collect keys to open doors"keys_bitmask
"Can use special ability X times"ability_count

📌Template: Cheapest Flights with K Stops

Python · Cheapest Flights, K Stops 
import heapq
from collections import defaultdict

def cheapest_flight(n, flights, src, dst, k):
    graph = defaultdict(list)
    for u, v, price in flights:
        graph[u].append((v, price))

    INF = float('inf')
    # dist[city][stops_used] = minimum cost
    dist = [[INF] * (k + 2) for _ in range(n)]
    dist[src][0] = 0
    heap = [(0, src, 0)]  # (cost, city, stops_used)

    while heap:
        cost, city, stops = heapq.heappop(heap)

        if city == dst:
            return cost

        if cost > dist[city][stops]:
            continue

        if stops + 1 > k + 1:
            continue

        for neighbor, price in graph[city]:
            new_cost = cost + price
            new_stops = stops + 1
            if new_cost < dist[neighbor][new_stops]:
                dist[neighbor][new_stops] = new_cost
                heapq.heappush(heap, (new_cost, neighbor, new_stops))

    return -1

📌Template: Grid with K Obstacle Removals (BFS)

Python · Shortest Path with K Obstacle Removals 
from collections import deque

def shortest_path_k_obstacles(grid, k):
    m, n = len(grid), len(grid[0])
    visited = [[[False]*(k+1) for _ in range(n)] for _ in range(m)]
    visited[0][0][k] = True
    queue = deque([(0, 0, k, 0)])
    dirs = [(0,1),(0,-1),(1,0),(-1,0)]

    while queue:
        r, c, rem, steps = queue.popleft()
        if r == m-1 and c == n-1:
            return steps
        for dr, dc in dirs:
            nr, nc = r+dr, c+dc
            if 0 <= nr < m and 0 <= nc < n:
                if grid[nr][nc] == 0 and not visited[nr][nc][rem]:
                    visited[nr][nc][rem] = True
                    queue.append((nr, nc, rem, steps+1))
                elif grid[nr][nc] == 1 and rem > 0 and not visited[nr][nc][rem-1]:
                    visited[nr][nc][rem-1] = True
                    queue.append((nr, nc, rem-1, steps+1))
    return -1

📌Template: Keys and Doors (Bitmask BFS)

Python · Bitmask BFS for Keys 
from collections import deque

def shortest_path_all_keys(grid):
    m, n = len(grid), len(grid[0])
    total_keys = 0
    start_r = start_c = 0

    for r in range(m):
        for c in range(n):
            if grid[r][c] == '@':
                start_r, start_c = r, c
            elif grid[r][c].islower():
                total_keys += 1

    target = (1 << total_keys) - 1
    visited = set()
    visited.add((start_r, start_c, 0))
    queue = deque([(start_r, start_c, 0, 0)])
    dirs = [(0,1),(0,-1),(1,0),(-1,0)]

    while queue:
        r, c, keys, steps = queue.popleft()
        for dr, dc in dirs:
            nr, nc = r+dr, c+dc
            if 0 <= nr < m and 0 <= nc < n:
                cell = grid[nr][nc]
                if cell == '#':
                    continue
                if cell.isupper() and not (keys & (1 << (ord(cell.lower())-ord('a')))):
                    continue
                new_keys = keys
                if cell.islower():
                    new_keys = keys | (1 << (ord(cell)-ord('a')))
                if new_keys == target:
                    return steps + 1
                if (nr, nc, new_keys) not in visited:
                    visited.add((nr, nc, new_keys))
                    queue.append((nr, nc, new_keys, steps+1))
    return -1
§5Variation 3

Multi-Source Dijkstra

03
Variation 3 · One-line change, huge impact
Multi-Source Dijkstra
All sources start at d=0 · seed the heap with every source

📌Problem Signal

Recognize this when you see…

"Distance to nearest X", "farthest from any X", "simultaneously from all".

📌Core Idea

Normal:       Push ONE source at dist=0
Multi-Source: Push ALL sources at dist=0

That's the ONLY change!

📌Template: Weighted Grid

Python · Multi-Source Dijkstra (weighted) 
import heapq

def multi_source_dijkstra(grid, cost, sources):
    m, n = len(grid), len(grid[0])
    INF = float('inf')
    dist = [[INF]*n for _ in range(m)]
    heap = []
    dirs = [(0,1),(0,-1),(1,0),(-1,0)]

    for r, c in sources:
        dist[r][c] = 0
        heapq.heappush(heap, (0, r, c))

    while heap:
        d, r, c = heapq.heappop(heap)
        if d > dist[r][c]:
            continue
        for dr, dc in dirs:
            nr, nc = r+dr, c+dc
            if 0 <= nr < m and 0 <= nc < n and grid[nr][nc] != '#':
                nd = d + cost[nr][nc]
                if nd < dist[nr][nc]:
                    dist[nr][nc] = nd
                    heapq.heappush(heap, (nd, nr, nc))
    return dist

📌Template: Unweighted BFS Version

Python · Multi-Source BFS (unweighted) 
from collections import deque

def multi_source_bfs(grid, sources):
    m, n = len(grid), len(grid[0])
    INF = float('inf')
    dist = [[INF]*n for _ in range(m)]
    queue = deque()
    dirs = [(0,1),(0,-1),(1,0),(-1,0)]

    for r, c in sources:
        dist[r][c] = 0
        queue.append((r, c))

    while queue:
        r, c = queue.popleft()
        for dr, dc in dirs:
            nr, nc = r+dr, c+dc
            if 0 <= nr < m and 0 <= nc < n and dist[nr][nc] == INF and grid[nr][nc] != '#':
                dist[nr][nc] = dist[r][c] + 1
                queue.append((nr, nc))
    return dist

📌With Nearest-Source Tracking

Python · Multi-Source + Nearest Source 
import heapq

def multi_source_with_tracking(grid, cost, sources):
    m, n = len(grid), len(grid[0])
    INF = float('inf')
    dist = [[INF]*n for _ in range(m)]
    nearest = [[None]*n for _ in range(m)]
    heap = []
    dirs = [(0,1),(0,-1),(1,0),(-1,0)]

    for r, c in sources:
        dist[r][c] = 0
        nearest[r][c] = (r, c)
        heapq.heappush(heap, (0, r, c))

    while heap:
        d, r, c = heapq.heappop(heap)
        if d > dist[r][c]:
            continue
        for dr, dc in dirs:
            nr, nc = r+dr, c+dc
            if 0 <= nr < m and 0 <= nc < n and grid[nr][nc] != '#':
                nd = d + cost[nr][nc]
                if nd < dist[nr][nc]:
                    dist[nr][nc] = nd
                    nearest[nr][nc] = nearest[r][c]
                    heapq.heappush(heap, (nd, nr, nc))

    return dist, nearest

§6Variation 4

0-1 BFS

04
Variation 4 · The clever optimization
0-1 BFS
Deque · push 0-cost to front, 1-cost to back · drops the log V factor

📌Problem Signal

Recognize this when you see…

Edge weights are only 0 and 1. Or reducible to 0 and 1.

📌Core Idea

Cost 0 neighbor → push to FRONT of deque
Cost 1 neighbor → push to BACK of deque

Deque stays sorted with at most 2 levels!
No heap needed → O(V+E) instead of O((V+E)logV)
Time (vs Dijkstra)
O(V + E)
Saved factor
log V

📌Template

Python · 0-1 BFS 
from collections import deque

def zero_one_bfs(graph, source):
    dist = {node: float('inf') for node in graph}
    dist[source] = 0
    dq = deque([source])

    while dq:
        u = dq.popleft()
        for v, w in graph[u]:
            new_dist = dist[u] + w
            if new_dist < dist[v]:
                dist[v] = new_dist
                if w == 0:
                    dq.appendleft(v)  # Free → FRONT
                else:
                    dq.append(v)      # Cost 1 → BACK
    return dist

📌Grid Template

Python · 0-1 BFS on Grid 
from collections import deque

def zero_one_bfs_grid(grid):
    m, n = len(grid), len(grid[0])
    INF = float('inf')
    dist = [[INF]*n for _ in range(m)]
    dist[0][0] = 0
    dq = deque([(0, 0)])
    dirs = [(0,1),(0,-1),(1,0),(-1,0)]

    while dq:
        r, c = dq.popleft()
        for dr, dc in dirs:
            nr, nc = r+dr, c+dc
            if 0 <= nr < m and 0 <= nc < n:
                cost = grid[nr][nc]  # 0 or 1
                nd = dist[r][c] + cost
                if nd < dist[nr][nc]:
                    dist[nr][nc] = nd
                    if cost == 0:
                        dq.appendleft((nr, nc))
                    else:
                        dq.append((nr, nc))

    return dist[m-1][n-1]

📌Edge-Weight Normalization Trick

When the weights aren't literally 0 and 1

Costs are X and X+1 (e.g., 2 and 3)?

  • Subtract X from all → costs become 0 and 1
  • Use 0-1 BFS for deque ordering
  • Track real distances using original weights

📌Common Mistake

Python · The Trap vs The Fix 
# ❌ WRONG: Using visited array
if visited[v]: continue

# ✅ CORRECT: Check distance improvement
if new_dist < dist[v]:  # A node CAN be updated multiple times
Why this matters

In 0-1 BFS, a node can be discovered through a cost-1 path first, then later improved through a cost-0 path. A naive visited set blocks that improvement and gives you a wrong answer. Always relax on distance, never on visited-flag.
